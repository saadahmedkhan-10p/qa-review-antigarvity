Role-Specific Permissions & Actions

Admin / QA Head
Responsible for system governance, project setup, and user management. Main driver of the configuration phase.
  • Manage Projects: Create, Edit, Delete, Close
  • Manage Users: Invite, Assign Roles, Remove
  • Define Forms: Create/Edit Assessment JSON forms
  • Reports: View & Export Full System Reports
  • Trigger Cycles: Initiate review cycles manually
Reviewer
Subject Matter Experts assigned to conduct technical assessments of projects.
  • Conduct Reviews: Fill out assigned forms
  • Schedule: Set planned review dates
  • View Projects: Access assigned project details
  • Comment: Add observations to reviews
  • Cannot delete or reassign reviews
QA Lead / Manager
Oversees the quality process, validates reviewer submissions, and ensures loop closure.
  • Monitor Status: Track overdue/pending reviews
  • Validate: Review submitted assessments
  • Feedback: Request changes or add feedback
  • Close Loop: Verified & Close reviews
  • Manage Forms: (Manager) Edit assessment criteria
Stakeholder (PM/Director)
Consumers of the QA insights for strategic decision making and risk analysis.
  • View Dashboard: High-level health metrics
  • Analyze Risks: Drill down into specific reports
  • Export Reports: (Director) Download PDF/Excel
  • Comment: Provide strategic input
  • Read-only access to configuration
